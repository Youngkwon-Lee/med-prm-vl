Med-PRM

Enhancing Medical Reasoning of LLMs
with Process Reward Models

RAG-AS-A-JUDGE를 활용한 의료 추론 검증

ETH Zürich Medical AI Lab | 2025

1. 연구 배경 및 문제

LLM 의료 추론의 한계

  • LLM은 의료 질문에서 "그럴듯하지만 틀린" 답변을 자주 생성
  • 최종 답만 평가하는 Outcome Reward Model (ORM)은 추론 과정의 오류를 놓침
  • 기존 PRM은 수학/코딩에 특화 → 의료 도메인에 부적합
핵심 문제: 의료 추론에서 "각 단계가 의학적으로 올바른가?"를 어떻게 검증할 것인가?

기존 접근법의 한계

ORM (Outcome RM)

  • 최종 답변만 평가
  • 잘못된 추론으로 맞은 답 → 높은 점수
  • 추론 과정 오류 감지 불가

PRM (Process RM)

  • 각 추론 단계를 개별 평가
  • 중간 단계 오류 감지 가능
  • 의료 도메인용 PRM 부재

2. Med-PRM 솔루션

핵심 아이디어: RAG-AS-A-JUDGE로 의료 문헌 기반 Step-wise 검증 → Medical PRM 학습

RAG-AS-A-JUDGE란?

각 추론 단계마다 관련 의학 문헌을 검색(RAG)하여, 해당 단계가 의학적으로 올바른지 자동으로 판단하는 방식

추론 Step
→
RAG 검색
→
문헌 기반 검증
→
+/- 레이블

주요 기여

  • RAG-AS-A-JUDGE: 의학 문헌 기반의 신뢰할 수 있는 Step-wise 레이블 생성
  • Med-PRM: 의료 도메인 최초의 Process Reward Model
  • MedQA 80.35%: 8B 모델 최초로 80% 돌파

3. 방법론: RAG-AS-A-JUDGE

Step-wise 검증 프로세스

1
Step 추출: 솔루션에서 각 추론 단계 분리
2
RAG 검색: 각 Step에 대해 관련 의학 문헌 검색 (MedCorp 사용)
3
Judge 평가: 검색된 문헌과 Step 내용 비교 → 의학적 정확성 판단
4
레이블 생성: 올바르면 +, 틀리면 - 레이블 부여

예시: COPD 진단 문제

Step 추론 내용 RAG 검색 결과 판정
1 40-pack-year 흡연력은 COPD 위험인자 "Smoking is primary risk factor for COPD" +
2 Flattened diaphragm은 폐 과팽창 징후 "Flattened diaphragms indicate hyperinflation" +

1단계: PRM 학습

  • Policy Model이 솔루션 생성 (CoT reasoning)
  • RAG-AS-A-JUDGE로 각 Step 검증 → [+, +, -, +, +] 형태의 레이블
  • 레이블된 데이터로 PRM 학습 (Binary Classification)

2단계: Policy Model 개선 (선택적)

  • Rejection Sampling: PRM 점수가 높은 솔루션만 선택
  • 선택된 고품질 솔루션으로 Policy Model SFT

5. PRM 학습 상세

학습 목표

각 추론 단계의 "ки" 마커 위치에서 + (correct) 또는 - (incorrect) 토큰의 확률을 예측

Loss = CrossEntropy( softmax([logit+, logit-]), [gt, 1-gt] )

학습 설정

파라미터 값
Base Model Llama-3.1-8B-Instruct
Learning Rate 2e-6
Epochs 3
Effective Batch Size 64 (1 × 64 accumulation)
Precision BFloat16
Max Sequence Length 4096 (RAG 포함 시)

6. Inference: Test-time Scaling

Best-of-N

N개 솔루션 생성 → PRM으로 각각 점수 계산 → Min Score가 가장 높은 솔루션 선택

N개 생성
→
PRM 점수
→
Best 선택

* Min Score: 모든 Step 중 가장 낮은 점수 (병목 품질)

SC+RM (Self-Consistency + RM)

N개 솔루션의 답변별 PRM 점수를 가중치로 Majority Voting

Score(answer) = Σ PRM_score(soli)

* 단순 다수결이 아닌 품질 가중 투표

Min Score 선택 이유

추론 체인에서 가장 약한 단계(병목)가 전체 품질을 결정합니다.
Min Score가 높다 = 모든 단계가 일정 수준 이상의 품질을 가짐

10. 핵심 인사이트

1. PRM > ORM in Medical Reasoning

의료 추론에서 Process Reward Model이 Outcome Reward Model보다 효과적

2. RAG-AS-A-JUDGE의 효용성

의학 문헌 기반 검증으로 신뢰할 수 있는 학습 데이터 생성 가능

3. Test-time Scaling 효과

N=32로 증가 시 MedQA 80% 돌파 (8B 모델 최초)

Why Min Score?

추론 체인은 가장 약한 고리(weakest link)에 의해 품질이 결정됨
→ Min Score가 높은 솔루션 = 모든 단계가 안정적인 품질

11. 한계점 및 향후 연구

한계점

  • Compute Cost: N=32 샘플링은 추론 시간 증가
  • RAG 의존성: 검색된 문헌의 품질에 따라 레이블 품질 변동
  • 도메인 특화: 의료 외 다른 도메인으로의 일반화 미검증

향후 연구 방향

  • 더 효율적인 Test-time Scaling 방법 연구
  • Multi-modal 의료 데이터 (영상, 병리) 확장
  • RAG 검색 품질 향상을 위한 의료 특화 검색 모델
  • 실제 임상 환경에서의 검증

Take-home Message

의료 AI에서 "최종 답"만 평가하는 것은 불충분합니다.
각 추론 단계의 의학적 정확성을 검증하는 것이 핵심이며,
RAG를 활용한 문헌 기반 검증이 이를 가능하게 합니다.
